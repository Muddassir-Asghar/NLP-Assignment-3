Part A: Encoder Model

We first preprocess the review text by lowercasing it, removing punctuation, tokenizing it into words, and building a vocabulary from the training split only. Each review is then converted into a fixed-length sequence using padding and truncation. Along with the original sentiment label derived from the star rating, we also define a second task: review length category, grouped into short, medium, and long based on the training-set tertiles.

The encoder is a Transformer-based multi-task model. It begins with an embedding layer and sinusoidal positional encoding so the model can represent both word meaning and word order. The sequence then passes through stacked encoder blocks, each containing multi-head self-attention, residual connections, layer normalization, and a position-wise feed-forward network. After the encoder stack, we apply masked mean pooling to obtain a single fixed-dimensional vector for the whole review. This vector is used by two classification heads: one for sentiment prediction and one for the derived length feature.

The encoder is trained with a combined loss, using cross-entropy for both tasks. After training, we save the best model checkpoint and export the learned review embeddings for the training and test sets. These embeddings are later reused by the retrieval module.

Part B: Retrieval Module

The retrieval stage uses the encoder’s learned embeddings as the search space. For each test review, we first obtain its encoder embedding, then compare it against all training embeddings using cosine similarity. The most similar training reviews are returned as the top-k retrieved results.

The retrieval module is designed to serve two roles. First, it acts as a semantic search mechanism that finds training examples similar to the current query. Second, it provides contextual evidence for generation in Part C. To evaluate retrieval quality, we measure whether retrieved neighbors share the same sentiment class and length category as the query, and we compare these scores across different values of k.

Part C: Decoder for Explanation Generation

The decoder is a decoder-only Transformer trained to generate a short natural-language explanation for each sentiment prediction. Its input is a structured prompt that combines four pieces of information: the original review, the predicted sentiment, the predicted derived feature, and the retrieved top-k context from Part B. This prompt is converted into a token sequence and used as the conditioning input for the language model.

The decoder uses masked causal self-attention, which ensures that each token can only attend to earlier tokens and not future ones. This is what makes the model autoregressive. Each decoder block contains causal self-attention, residual connections, layer normalization, and a feed-forward network. The model is trained with a language-model objective to predict the next token in the explanation sequence. During evaluation, we report perplexity on the held-out test set.

To show the benefit of retrieval, we train two versions of the decoder: a full RAG model that includes retrieved context in the input prompt, and a baseline model that does not use retrieval. We then compare their test perplexity and generate at least five example explanations from both systems. This lets us judge both quantitative performance and qualitative coherence.
